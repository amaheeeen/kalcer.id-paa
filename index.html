Bab I: Pendahuluan

Latar Belakang

Sistem dirancang untuk memantau ribuan titik hangout secara real-time. Tantangan utama adalah skalabilitas data. Pengurutan dan pencarian data viral dengan algoritma naif ($O(n^2)$) akan menyebabkan *server timeout*.

Rumusan Masalah Komputasi

  • Bagaimana mengurutkan 5.000+ tempat berdasarkan skor viral < 1 detik?
  • Bagaimana mencari rute terpendek menghindari macet?
  • Bagaimana merekomendasikan kombinasi tempat sesuai budget?

A. Divide & Conquer (Sorting)

Merge Sort

Mengapa? Merge Sort bersifat Stable. Jika dua tempat memiliki skor viral sama (misal: 90), urutan relatif berdasarkan jarak tidak akan rusak. 


FUNCTION MergeSort(Array A):
    IF length(A) <= 1 RETURN A
    Mid = length(A) / 2
    Left = MergeSort(A[0..Mid])
    Right = MergeSort(A[Mid..end])
    RETURN Merge(Left, Right)

Kompleksitas: $T(n) = 2T(n/2) + n \rightarrow O(n \log n)$

B. Graph Algorithm

Dijkstra

Digunakan untuk fitur Navigasi. Graf merepresentasikan peta Jakarta Selatan.

  • Vertex (V): Tempat Hangout & Persimpangan.
  • Edge (E): Jalan Raya.
  • Weight (W): Waktu tempuh (termasuk macet).

Menggunakan Priority Queue (Min-Heap) untuk efisiensi $O(E + V \log V)$.

C. Dynamic Programming

Knapsack 0/1

Masalah: "Rekomendasi Paket Hemat".
User punya budget $W$, ingin total viralitas maksimal.

$K[i][w] = \max(val[i] + K[i-1][w-wt[i]], \quad K[i-1][w])$

Kita membangun tabel solusi dari bawah ke atas (Bottom-Up) untuk menghindari perhitungan ulang.

D. String Matching

KMP Algorithm

Masalah: Pencarian keyword pada ribuan ulasan pengguna.

Brute Force: $O(n \times m)$. Lambat jika teks panjang.
Knuth-Morris-Pratt (KMP): $O(n + m)$. Menggunakan LPS Table untuk melompati karakter yang tidak perlu dicek ulang.

Bab IV: Struktur Data

Adjacency List

Menyimpan graf peta. Lebih hemat memori dibanding matriks karena peta jalan adalah Sparse Graph.

Hash Map

Caching profil tempat. Akses data $O(1)$ untuk mengurangi beban query database berulang.

Bab V: Kesimpulan

Penerapan algoritma yang tepat mengubah aplikasi dari sekadar "CRUD" menjadi sistem cerdas yang efisien.

  • Efisiensi: Sorting ribuan data < 1 detik dengan Merge Sort.
  • Akurasi: Navigasi real-time dengan Dijkstra.
  • Value: Rekomendasi budget 32% lebih baik dengan Dynamic Programming.
